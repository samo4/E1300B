I-SCPI Drivers

I-SCPI is not recommended for new designs because I-SCPI is not supported on new hardware and current Agilent IO Libraries.

For background information about I-SCPI (Interpreted Standard Commands for Programmable Instruments), see About Instrument Drivers.

Register-based VXI modules installed in FireWire (E8491), Embedded VXI Controllers (E623x, E985x, EPC-7, EPC-8), MXI, VXLink systems (E1383, E1483), or National Instruments MXI-2 require an I-SCPI driver to provide programming capability. This information does not apply to message-based modules which have built-in SCPI drivers.

This page contains links to:

  • I-SCPI Components: necessary for use with Agilent I/O Libraries (used with the I-SCPI VISA I/O drivers).
  • I-SCPI VISA Drivers: Agilent I-SCPI Instrument Drivers that use VISA I/O (these are the most recent).
  • I-SCPI SICL Drivers: Agilent I-SCPI Instrument Drivers that use SICL I/O (these are the previous versions).

I-SCPI Components

The I-SCPI components are necessary to use I-SCPI with current Agilent I/O Libraries. These are intended for use with the most recent versions of I-SCPI that work with VISA I/O.

The I-SCPI components are contained in a self-extracting archive, iscpi_io_components.exe, that should be unzipped into the I/O Libraries installation directory, typically C:\Program Files\Agilent\IO Libraries. If you have placed the IO Libraries in some other directory, be sure to use this other directory.

I-SCPI VISA Drivers

These I-SCPI drivers work with VISA I/O and are associated with the G.02.00 and later releases of the Agilent IO Libraries.

All of these I-SCPI VISA drivers are collected in a self-extracting archive, all_visa_iscpi_drivers.exe, that should be unzipped into the VXIplug&play BIN directory, typically C:\Program Files\VISA\windir\BIN or C:\VXIPNP\windir\BIN, where windir is WINNT for NT/2000/XP or WIN95 for 95/98/Me. If you have placed the VXIplug&play BIN in a different place, be sure to use that other directory.
